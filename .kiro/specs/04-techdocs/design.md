# Design Document

## Overview

This feature enables TechDocs in-app documentation for the Backstage portal demo by switching the generator from Docker to local mode, installing the required Python dependency in the Dockerfile, adding MkDocs source files for the `backstage-portal` component, and annotating its catalog entity so the Docs tab appears.

No new Backstage plugins are needed — TechDocs is already installed in the backend. This is purely configuration and file additions.

## Architecture

```
User clicks "Docs" tab
        │
        ▼
TechDocs Frontend Plugin
        │  requests rendered docs
        ▼
TechDocs Backend Plugin
        │  checks local cache (techdocs-cache dir)
        │  if not cached → triggers generation
        ▼
Local Generator (mkdocs CLI)
        │  reads mkdocs.yml + docs/ from entity source dir
        │  runs: mkdocs build
        ▼
Local Publisher
        │  stores output in container filesystem
        ▼
TechDocs Frontend Plugin renders HTML
```

The `techdocs-ref: dir:.` annotation tells the backend to look for `mkdocs.yml` relative to the `catalog-info.yaml` file location. Since `catalog-info.yaml` is at `backstage-portal/catalog-info.yaml`, the docs source is `backstage-portal/` (i.e., `mkdocs.yml` at `backstage-portal/mkdocs.yml` and `docs/` at `backstage-portal/docs/`).

## Components

### 1. `app-config.yaml` — Generator Mode Change

Change `techdocs.generator.runIn` from `docker` to `local`. The rest of the TechDocs config (`builder: local`, `publisher.type: local`) is already correct.

Before:
```yaml
techdocs:
  builder: 'local'
  generator:
    runIn: 'docker'
  publisher:
    type: 'local'
```

After:
```yaml
techdocs:
  builder: 'local'
  generator:
    runIn: 'local'
  publisher:
    type: 'local'
```

### 2. `packages/backend/Dockerfile` — Python Dependency

Add a `pip install mkdocs-techdocs-core` instruction. This must run as root (before any `USER` switch) so the package is available system-wide.

The install line goes after the Node.js base image setup and before the `COPY` / `CMD` instructions:

```dockerfile
RUN pip3 install mkdocs-techdocs-core --break-system-packages
```

The `--break-system-packages` flag is required on Debian Bookworm+ (Python 3.11+) where pip enforces externally-managed environments. Alternatively, a virtual environment can be used, but the flag is simpler for a demo image.

### 3. `backstage-portal/mkdocs.yml` — MkDocs Config

New file at the monorepo root (alongside `catalog-info.yaml`):

```yaml
site_name: Backstage Portal
docs_dir: docs
plugins:
  - techdocs-core
```

### 4. `backstage-portal/docs/index.md` — Documentation Content

New file with minimal but useful content describing the portal:

```markdown
# Backstage Portal

Welcome to the Backstage developer portal documentation.

## Overview

This portal is built with [Backstage](https://backstage.io) and provides:
- Software catalog
- TechDocs in-app documentation
- Software templates (scaffolder)
- GitHub authentication

## Local Development

Run from `backstage-portal/`:

    yarn start

The app runs on http://localhost:3000 and the backend on http://localhost:7007.
```

### 5. `backstage-portal/catalog-info.yaml` — techdocs-ref Annotation

Add the `backstage.io/techdocs-ref: dir:.` annotation to the existing `backstage-portal` component entity:

```yaml
metadata:
  name: backstage-portal
  description: An example of a Backstage application.
  annotations:
    backstage.io/techdocs-ref: dir:.
```

## File Change Summary

| File | Change |
|------|--------|
| `backstage-portal/app-config.yaml` | `generator.runIn: docker` → `local` |
| `backstage-portal/packages/backend/Dockerfile` | Add `pip3 install mkdocs-techdocs-core` |
| `backstage-portal/catalog-info.yaml` | Add `backstage.io/techdocs-ref: dir:.` annotation |
| `backstage-portal/mkdocs.yml` | New file — MkDocs config |
| `backstage-portal/docs/index.md` | New file — documentation content |

## Correctness Properties

1. Round-trip: docs generated by `mkdocs build` using `mkdocs.yml` + `docs/index.md` produce valid HTML that TechDocs can serve — verifiable by triggering the Docs tab and confirming no 500 error.
2. Annotation presence: `catalog-info.yaml` contains `backstage.io/techdocs-ref` — verifiable by `grep` or YAML parse.
3. Generator mode: `app-config.yaml` `techdocs.generator.runIn` equals `local` — verifiable by config parse.
4. Dependency installed: `pip show mkdocs-techdocs-core` inside the built container exits 0.

## Out of Scope

- S3 / GCS / external storage
- CI-based pre-generation
- Docker-in-Docker
- TechDocs for any entity other than `backstage-portal`
- Custom MkDocs themes or extensions beyond `techdocs-core`
