# Claude Code Instructions

## Git workflow

- Always work on a feature branch (never commit directly to `main`)
- When merging to main, **create a PR** — never `git merge` directly
- Use `gh pr create` to open the PR, then share the URL with the user
- Branch naming: `feature/<spec-number>-<short-description>`

## Terraform

- Always run from `terraform/` directory
- Source credentials before apply: `source .env.prod && AWS_PROFILE=chile terraform apply`
- Never commit `terraform.tfvars` or `.env*` files

## AWS

- Always use `--profile chile` for AWS CLI commands
- Region: `us-east-1`

## Specs

- Follow the Kiro spec-driven approach: create spec first (requirements + design + tasks), implement after
- Update task checkboxes in `tasks.md` as work completes
- Spec directory: `.kiro/specs/<number>-<name>/`

## Deployment Context for Future Reference

### Infrastructure Overview

The Backstage portal is deployed on AWS using:
- **VPC**: Custom VPC (10.0.0.0/16) with public/private subnets across 2 AZs (us-east-1a, us-east-1b)
- **Compute**: ECS Fargate (cluster: `backstage-mvp-cluster`, service: `backstage-mvp-service`)
- **Database**: RDS PostgreSQL in private subnets
- **Load Balancer**: Application Load Balancer with HTTPS (ACM certificate)
- **Container Registry**: ECR repository `backstage-mvp`
- **DNS**: Route53 hosted zone for `backstage.glaciar.org`
- **Networking**: VPC endpoints (no NAT Gateway) for cost optimization
- **CI/CD**: GitHub Actions with OIDC (no long-lived credentials)

### Key Terraform Outputs

After `terraform apply`, these outputs are available:

```bash
terraform output portal_nameservers    # NS records for subdomain delegation
terraform output alb_dns_name          # ALB DNS for Route53 alias
terraform output ecr_repository_url    # ECR URL for docker push
terraform output rds_endpoint          # Database endpoint
terraform output ecs_cluster_name      # ECS cluster name
terraform output github_actions_role_arn  # IAM role ARN for GitHub Actions
```

#### GitHub Actions (stored as repository secrets)
- `AWS_ROLE_ARN` — IAM role ARN from terraform output `github_actions_role_arn`

#### ECS Task Environment Variables (managed by Terraform via Secrets Manager)
- `APP_BASE_URL` — https://backstage.glaciar.org
- `POSTGRES_HOST` — RDS endpoint
- `POSTGRES_PORT` — 5432
- `POSTGRES_USER` — backstage
- `POSTGRES_PASSWORD` — (generated by Terraform)
- `AUTH_GITHUB_CLIENT_ID` — GitHub OAuth client ID
- `AUTH_GITHUB_CLIENT_SECRET` — GitHub OAuth client secret
- `GITHUB_TOKEN` — GitHub PAT for scaffolder

### Deployment Flow

1. **Push to main** triggers `.github/workflows/deploy.yml`
2. **GitHub Actions**:
   - Authenticates to AWS via OIDC (no access keys)
   - Builds backend: `yarn build:backend`
   - Builds Docker image from `packages/backend/Dockerfile`
   - Pushes to ECR with tags: `<sha>` and `latest`
   - Updates ECS task definition with new image
   - Deploys to ECS and waits for service stability
3. **ECS** pulls image from ECR and runs on Fargate
4. **ALB** routes traffic to ECS tasks
5. **Route53** resolves `backstage.glaciar.org` to ALB

### Manual Deployment

If you need to deploy manually:

```bash
cd backstage-portal

# Build
yarn build:backend

# Build Docker image
docker build -f packages/backend/Dockerfile -t backstage:local .

# Tag and push to ECR
aws ecr get-login-password --region us-east-1 --profile chile | \
  docker login --username AWS --password-stdin <ecr-url>
docker tag backstage:local <ecr-url>:manual
docker push <ecr-url>:manual

# Update ECS service
aws ecs update-service \
  --cluster backstage-mvp-cluster \
  --service backstage-mvp-service \
  --force-new-deployment \
  --profile chile \
  --region us-east-1
```

### Cost Optimization Notes

- Using VPC endpoints instead of NAT Gateway saves ~$28/month
- Single AZ NAT (if needed) instead of multi-AZ saves ~$32/month
- Current estimated cost: ~$60/month (see `terraform/README_COSTS.md`)

### Domain Configuration

The portal runs at `backstage.glaciar.org`:
1. Route53 hosted zone created by Terraform
2. NS records must be added to parent domain (`glaciar.org`) registrar
3. ACM certificate auto-validates via DNS
4. ALB HTTPS listener uses the certificate

### Troubleshooting

**ECS tasks failing to start:**
- Check CloudWatch logs: `/ecs/backstage-mvp-backstage`
- Verify secrets in Secrets Manager
- Ensure ECR image exists and is accessible

**Database connection issues:**
- Verify RDS security group allows ECS task security group
- Check VPC endpoints for Secrets Manager are working
- Validate connection string in task definition

**GitHub OAuth not working:**
- Verify callback URL in GitHub OAuth app: `https://backstage.glaciar.org/api/auth/github/handler/frame`
- Check secrets in Secrets Manager match GitHub OAuth app

**DNS not resolving:**
- Verify NS records in parent domain registrar
- Check Route53 hosted zone has correct A record alias to ALB
- Wait for DNS propagation (up to 48 hours)
